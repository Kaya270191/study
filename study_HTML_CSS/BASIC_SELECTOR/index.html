개발자 취업

회사가 원하는 것?

  • 기술력 - 현재 얼마나 준비되었나?
  • 성장성 - 앞으로 더욱 발전할까?
  • 팀워크 - 함께 일하기 좋은가?

준비해야 할 것?

  • 자소서 - 나를 회사에 파는 광고, 좋은 광고란?
  • 포트폴리오 - 직접 만든 무언가. 실제 물건은 어떤가?
  • 코딩 테스트 - 직접적 실력 검증. 온라인 및 화이트 보드 테스트

자격증 준비해야 할까요?

굳이 필요x 직접 회사에서 언급 시에만 준비

개발자 공부법

개념

  • 큰 그림부터- 목차부터, 핵심부터, 사소한 것 제껴
  • 얕고 빠르게- 한 번에 다X, 뼈대부터 먼저
  • 점진적 확장- 뼈대에 살을 붙이기

코드

  • 따라치기-기초가 곧 핵심. 어려운 것 스킵
  • 바꿔보기-내 입맛대로 변경. 가설->검정
  • 출제하기-출제자의 입장

문서화

  1. why - 개발은 팀 단위. 원활한 협업을 위해
  2. how - 문학적x, 논리적, 간결
  3. what - 블로그(부수입), 깃허브(취업 시 가산점)
